# AI usage guide

The canonical, always-loaded AI guide lives at the repository root:
**[`CLAUDE.md`](../../CLAUDE.md)**.

It documents:

- why Lily UI is structured to be AI-friendly,
- the component API contract and the shared prop vocabulary
  (`status` / `size` / `variant` / `block`),
- the `.lily-*` class-name scheme,
- a tokens cheat-sheet (use semantic tokens, never raw values),
- theming & density attributes,
- do/don't rules for generated code.

For the visual worldview those rules express, see
[design-language.md](design-language.md).
